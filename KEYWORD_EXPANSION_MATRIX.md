## 4. Priority Score System

### 4.1 Scoring Formula

```
Priority Score = (Search Demand × 0.30) + (Buyer Intent × 0.30) + (Affiliate Value × 0.20) + (Competition Inverse × 0.20)
```

Each component is scored 0-100:

| Component | Weight | Scoring Guidance |
|---|---|---|
| Search Demand | 30% | Based on estimated monthly search volume. 100 = very high volume (>50k/mo), 50 = moderate (5k-10k), 0 = negligible |
| Buyer Intent | 30% | 100 = transactional ("buy", "best", "review"), 50 = commercial investigation ("vs", "comparison"), 20 = informational |
| Affiliate Value | 20% | 100 = high-ticket Amazon product ($300+), 50 = mid-ticket ($50-300), 20 = subscription software, 0 = free/no affiliate |
| Competition (Inverse) | 20% | 100 = very low competition (long-tail, niche), 50 = moderate, 0 = dominated by major publishers |

### 4.2 Tier Classification

| Tier | Score Range | Action |
|---|---|---|
| Tier 1 | 80-100 | Build immediately — highest ROI |
| Tier 2 | 60-80 | Build in second wave — solid ROI |
| Tier 3 | <60 | Build last or skip — low ROI |
